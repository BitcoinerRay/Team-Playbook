# Project Classification Principles

## Greenfield vs Brownfield

- **Greenfield**: New codebase. Full freedom to apply standards. Use presets as starting point.
- **Brownfield**: Existing codebase. Incremental adoption. Prioritize high-impact, low-friction modules. Avoid breaking changes.

## Product vs Platform

- **Product**: User-facing, outcome-driven. Emphasize PRD, launch, and product-thinking principles.
- **Platform**: Internal or developer-facing. Emphasize architecture, API design, and stability.

## Risk Tiers

- **High**: Security, payments, compliance. Stricter review, more documentation, explicit rollback plans.
- **Medium**: Core features, integrations. Standard process.
- **Low**: Experiments, internal tools. Lighter process, faster iteration.

Classify early. Adjust process and preset selection accordingly.
