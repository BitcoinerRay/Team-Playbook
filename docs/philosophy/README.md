# Philosophy

## Why Modular Knowledge, Not a Doc Heap

This repository is designed as a **modular knowledge system**, not a collection of loose markdown files.

- **Composability**: Modules (rules, skills, commands, templates) can be combined via presets. New projects pick what they need.
- **Traceability**: Each module has metadata (id, owner, version, status). Changes are auditable.
- **Consumption**: AI agents and automation tools parse structured content. Loose docs are hard to consume programmatically.
- **Maintenance**: Atomic modules age independently. Deprecating one does not break others.

The alternative—a flat doc heap—leads to duplication, drift, and low reuse. We invest in structure so the team can scale knowledge without scaling confusion.
